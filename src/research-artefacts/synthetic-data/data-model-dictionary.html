Synthetic Data Model and Data Dictionary

Synthetic Data Research Artefact
Synthetic Data Data Model Clinical Simulation

Overview

This document defines the conceptual entities and relationships for the synthetic health data generated to support clinical simulation and evaluation. It focuses on human-readable definitions and example values to ensure clinical face validity without providing machine-readable schemas.

Entities and Attributes

1. Patient Profile

Represents the demographic and baseline clinical characteristics of a synthetic individual.

Attribute Description Constraints Example Values
Synthetic ID A unique non-identifiable reference for the agent. Alphanumeric, unique within simulation scope. SYN-PT-8821, PT-BPP-001
Age Range The demographic age bracket for the scenario. Must align with disease prevalence assumptions. 45-55, 65-70
Gender Identity Self-identified gender for social context modelling. Discrete categories. Male, Female, Non-binary
Baseline BP The initial blood pressure state before any interventions. Systolic (90-200), Diastolic (60-120). 145/95 mmHg
Comorbidities Narrative description of existing conditions. Textual list of clinical states. Type 2 Diabetes, Mild Asthma

2. Clinical Encounter

Represents a point-in-time interaction between a patient and a healthcare provider or AI agent.

Attribute Description Constraints Example Values
Encounter Type The setting of the interaction. GP visit, A&E, Virtual triage. Triage Assessment
Symptom Narrative A human-readable description of presenting symptoms. Prose describing patient complaints. "Patient reports mild chest tightness for 2 hours."
Vital Signs Snapshot of physiological metrics at time of encounter. Clinically plausible ranges. HR: 82, SpO2: 98%

3. Medication Event

Represents the administration or prescription of a drug within the simulation.

Attribute Description Constraints Example Values
Drug Category The class of medication being simulated. Non-proprietary generic names. ACE Inhibitor, Beta Blocker
Dosage Logic Qualitative description of the dosage level. Low, Standard, High. Standard dose (e.g., 5mg daily)

Relationship Diagrams

Patient Journey Concept

  • Patient -> triggers -> Encounter
  • Encounter -> results in -> Observation
  • Observation -> leads to -> Medication Event

Example Data Record (Narrative)

Patient SYN-PT-105 is a 52-year-old male with a baseline BP of 150/95. During a routine hypertension screening encounter, he reports occasional headaches. The AI agent recommends starting a low-dose ACE inhibitor.
